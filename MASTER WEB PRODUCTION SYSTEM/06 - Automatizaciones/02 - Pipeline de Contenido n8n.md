# 02 — Pipeline de Contenido n8n

> ⚠️ HUECO: no implementado — diseño propuesto.

> Propósito: diseño objetivo de un flujo n8n que genere y publique artículos de blog `.mdx` contra el esquema del Vault. **No existe en ningún repo.** Hoy el blog se produce a mano.

## Estado real (verificado en auditoría)

**n8n NO existe en código en ninguno de los 31 proyectos.** Lo que hay es preparación, no integración:

- **GAMADEMEXICO:** el schema de blog dice estar "optimizado para automatización n8n" y hay helpers `generateMarkdownFrontmatter`/`validateBlogFrontmatter` — pero **no hay workflow, endpoint ni credencial n8n** en el repo. Es una convención de formato preparada para un n8n externo. Ver [[../_AUDITORIA/diagnostico-GAMADEMEXICO]] §HUECOS.
- **INFLAPY:** la infraestructura está **lista** pero el workflow no: campos Zod `autogenerated` (default false) y `audit` (`source`/`aiModel`/`workflowVersion`), contrato documentado en `docs/article-system-n8n.md`, y **1 artículo demo** (`ejemplo-articulo-automatizado-n8n.mdx`, `draft+noindex`, `audit.source:n8n`). **Sin webhook ni código n8n.** La auditoría lo clasifica como "🤖 Automatizable — solo falta conectar el workflow n8n real". Ver [[../_AUDITORIA/diagnostico-INFLAPY]] §Integraciones.

Mientras tanto, el blog se produce con el **proceso editorial manual** de [[05 - Produccion de Contenido/02 - Blog y Articulos SEO]] (investigación → redacción → `validate:mdx` → interlinking → anti-canibalización).

## Diseño objetivo (propuesta, a construir)

Un workflow n8n que produzca `.mdx` válidos para la colección `articulos`, respetando todas las reglas del Vault. **El contrato de salida es el frontmatter Zod `.strict()` de [[08 - Biblioteca Plantillas/_scaffold/content.config.ts|content.config.ts]]** — ese es el punto de integración real ya existente.

### Flujo propuesto

```
1. Trigger        → keyword/tema desde una cola (Sheet, webhook, calendario editorial)
2. Investigación  → SERP / keyword intent (NeuronWriter ya se usó como herramienta de proceso en GAMADEMEXICO)
3. Generación     → redacción del cuerpo .mdx (tono experto de campo, estructura del TEMPLATE-ARTICULO)
4. Imágenes       → handoff al pipeline fal.ai → AVIF + alt  (ver [[03 - Generacion de Imagenes fal.ai]], también HUECO)
5. Frontmatter    → arma el frontmatter contra el esquema `articulos`:
                     title ≤70 · description 70-160 · category ∈ ARTICLE_CATEGORIES (enum cerrado)
                     heroImage /images/… · pubDate · author · tags ≤10
                     relatedProducts/relatedServices/relatedPosts (reference por slug)
                     autogenerated: true · audit: {source:'n8n', aiModel, workflowVersion}
6. Commit / PR    → escribe src/content/articulos/<slug>.mdx en una rama y abre PR
7. Gate (CI)      → astro check + validate:mdx + lint SEO + link-check  (ver [[01 - Scripts Reales]])
                     el artículo NO se publica si no pasa el gate
8. Merge → deploy → Cloudflare Pages
```

### Por qué este diseño encaja con lo real

- **El contrato ya existe:** los campos `autogenerated`/`audit` de INFLAPY y el frontmatter `.strict()` del Vault son exactamente la interfaz que n8n debe llenar. n8n no inventa formato; rellena el esquema que ya valida el build.
- **El gate ya existe:** `validate:mdx` + `astro check` (REAL, [[01 - Scripts Reales]]) son la red de seguridad. Contenido generado que viole el esquema **rompe el build** — la automatización no puede degradar la calidad.
- **Publicación por PR, no directa:** el artículo entra como borrador (`draft`/`noindex`) y pasa por gate + revisión, como el demo de INFLAPY. Calidad > velocidad ([[05 - Produccion de Contenido/02 - Blog y Articulos SEO]]).

### Qué falta para que sea real

1. La instancia n8n (host, credenciales) — **no existe**.
2. El nodo de generación (modelo + prompt del tono de marca) — **no existe**.
3. El nodo de commit/PR a GitHub — **no existe**.
4. Conexión con el pipeline de imágenes ([[03 - Generacion de Imagenes fal.ai]], también HUECO).

Hasta tener esos cuatro, esto es **diseño**, no función. SOP relacionado: [[SOP 09 - Automatizacion de contenido]].
